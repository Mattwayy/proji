# Гайдлайн стиля «рабочих» AI-компонентов

> Документ фиксирует визуальный паттерн, к которому привели AI-элементы интерфейса
> (PROGPT) после правок 2026-06-22, чтобы новые экраны/компоненты не расходились
> по стилю и не возвращались к «скелетон»-заглушкам. Конвенция как у других
> рабочих документов в репо (`BACKEND_TASKS.md`, `ROADMAP_NEXT_STAGES.md`) —
> рабочий список правил, а не отчёт.

---

## 1. Плавающая кнопка «Спросить PROGPT» (`src/components/AIBadge.tsx`)

Глобальный триггер AI-панели, висит `fixed bottom-* right-*` на всех страницах.

- Базовое состояние: тёмная капсула `bg-slate-900 text-white` (НЕ светлая/белая —
  раньше была бледная пилюля с подписью "AI", это и резало глаз на скриншотах).
- Активное состояние (`isAiPanelOpen`): `bg-proji-primary text-white`.
- Состав: иконка `Sparkles` слева → текст `text-xs font-black uppercase tracking-wide`
  ("СПРОСИТЬ PROGPT") → опционально индикатор `isProcessing` → круглая иконка
  микрофона `Mic` в полупрозрачном кружке справа (`bg-white/15 rounded-full`).
- Обязательные `aria-label`/`aria-expanded` (доступность).

Если где-то понадобится новый AI-триггер (не глобальный, а локальный, на странице) —
используй тот же паттерн «тёмная капсула + Sparkles + подпись + Mic-кружок»,
а не изобретай новый стиль.

## 2. Чипы AI-подсказок (suggestion chips)

Используются в `/messages` (`SUGGESTIONS`), могут появиться в других местах
(чат, контекстная панель).

- Обычный чип: `bg-white border border-slate-200 text-slate-700 text-sm font-bold
  px-4 py-2 rounded-full`. НЕ тонкие бледные чипы — `text-xs` + `border/20` слишком
  слабые, на скриншотах выглядят как недогруженный UI.
- Последний/«контекстный» чип (тот, что реально дёргает AI по контексту,
  а не просто быстрый ответ) — визуально выделяется: `bg-gradient-to-r
  from-indigo-600 to-violet-600 text-white shadow-md shadow-indigo-600/25`
  + иконка `Sparkles` слева от текста.
- Чипы — `overflow-x-auto` в одну строку, `shrink-0` на каждом, чтобы не схлопывались.

## 3. Анимированная «glow»-рамка инпута (AI input bar)

Используется в `app/chat/page.tsx` (`.ai-input-glow`) и
`src/components/ContextSidebar.tsx` (`.ctx-input-glow`).

**Правило: НЕ использовать вращающийся `conic-gradient` (`::before` + `animation:
rotate`) на вытянутых/пилюле-образных контейнерах.** Эта техника работает только
для круглых/квадратных элементов — на широком input bar она даёт жёсткий
диагональный артефакт вместо тонкой рамки (баг, исправленный в этой сессии).

Правильный паттерн — скользящий `linear-gradient`:

```css
.xxx-input-glow {
  background: linear-gradient(90deg, #3b82f6, #14b8a6, #6366f1, #3b82f6);
  background-size: 300% 100%;
  animation: xxx-glow-move 5s linear infinite;
}
@keyframes xxx-glow-move {
  to { background-position: -300% 0; }
}
```

Обёртка: `relative rounded-full (или rounded-2xl) p-[1.5px] overflow-hidden` +
внутренний `div` с непрозрачным фоном (`bg-white`), который маскирует всё,
кроме тонкой 1.5px рамки.

## 4. Контент вместо «скелетонов»

При наполнении карточек/иллюстраций — НЕ оставлять голые серые полоски
(`bg-slate-100/200 rounded-full`, имитирующие текст) как единственный контент.
Хотя бы часть строк должна быть настоящим читаемым текстом (название файла,
задачи, метрика, статус) — иначе на скриншотах компонент выглядит «недоделанным»,
даже если технически это просто placeholder-стиль.
Пример до/после — `app/login/illustrations.tsx`:
`IllustrationQuickCreate`, `IllustrationContextHelp`.

## 5. Чек-лист перед тем как считать AI-компонент готовым

- [ ] Нет голых серых скелетон-полосок без подписи рядом — либо реальный текст,
      либо текст + полоска как второстепенный элемент.
- [ ] Если есть анимированная рамка/обводка — проверено визуально на реальной
      форме контейнера (пилюля ≠ круг, шаблон conic-gradient сюда не подходит).
- [ ] Кнопки/чипы с AI-функцией — тёмная капсула или градиентная заливка
      (`indigo-600 → violet-600`), не бледный/еле заметный стиль.
- [ ] `aria-label` на всех новых интерактивных элементах (см. `ROADMAP_NEXT_STAGES.md`,
      раздел 1).

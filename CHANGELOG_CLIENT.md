# Журнал изменений Proji — для заказчика

---

## Версия 0.4 — текущая сборка (май 2025)

### ✅ Выполнено сегодня

#### Панель администратора
- Новая страница `/admin` — рассылка задач сотрудникам по отделам
- Администратор выбирает: отдел-получатель (один или все), приоритет, срок, описание
- Задачи видны сотрудникам в своих доменных страницах (Юридический, Маркетинг и др.)
- Архивирование и удаление разосланных задач

#### Юридический дашборд (полностью реализован)
- Таблица договоров: статус, контрагент, тип, срок, уровень риска
- Судебные дела с инстанциями и результатами
- Фильтрация и поиск по договорам
- Раздел «Задачи от руководства» — задачи, разосланные администратором

#### Управленческие страницы
- **Отчёт**: KPI-дашборд (выручка, прибыль, NPS, OKR-выполнение) с прогресс-барами и трендами
- **Отчёт**: Сводка по отделам — выполненные и незакрытые задачи, уровень риска
- **Отчёт**: Лента последних отчётов сотрудников
- **Журнал**: Полноценный менеджер заметок — создание, категории (Решение/Идея/Риск/Встреча/Общее), закрепление, поиск, удаление

#### Маркетинговые страницы
- **Кампании**: таблица рекламных кампаний с бюджетом, показами, CTR, лидами; фильтр по каналу; прогресс-бар расхода бюджета
- **Лиды**: CRM с таблицей и Kanban-видом; pipeline-воронка; скоринг; статусы от «Новый» до «Выиграно/Проигран»
- **SEO**: позиции ключевых слов, динамика (+/-), сложность запроса, видимость сайта (график), топ страниц

#### API и архитектура
- Создан слой `src/lib/api.ts` — все операции с данными вынесены в одно место
- Каждая функция помечена комментарием с будущим REST-эндпоинтом
- Фронтенд готов к подключению бэкенда без изменений в компонентах

#### Исправленные баги
- Имена проектов теряются при обновлении страницы → проекты теперь сохраняются в localStorage и восстанавливаются при старте
- Выполненные задачи не отображались в форме отчёта → исправлено чтение из правильного источника данных
- Страница отчётов: форма теперь скрывается после отправки, отчёт появляется в истории

## Версия 0.3 — прошлая сборка

### ✅ Что было реализовано

#### Система задач проекта
- Задачи хранятся в localStorage отдельно от Zustand (не сбрасываются при навигации)
- Полный жизненный цикл: Новая → Принята → На проверке / Выполнена / Отклонена
- Модальное окно задачи: заголовок, описание, приоритет, дедлайн, вложения
- Кнопки по статусу: Принять / Отказать / Сдать на проверку / Завершить успешно
- Причины отказа — 7 вариантов радио-кнопок + произвольный комментарий
- «Получить задачи» — загружает 3 тестовые задачи от тимлида

#### Рабочее пространство проекта
- Обзор проекта: прогресс, статус, бюджет, команда, дедлайн
- Три раздела (Задачи / Отчёты / Документы) — каждый открывается отдельной полноэкранной страницей
- Навигация breadcrumb в топбаре показывает имя проекта вместо ID

#### Отчёты по проектам
- Таймер рабочего дня (Play/Pause/Reset) с сохранением в localStorage
- Выбор выполненных задач для включения в отчёт
- Текстовый комментарий к отчёту
- Отправка — сохраняет отчёт, показывает подтверждение
- История отправленных отчётов

#### Документы проекта
- Создание, редактирование, удаление документов
- Базовое форматирование: Жирный / Курсив / Подчёркнутый
- Экспорт в `.doc` формат (Word-совместимый)
- Поиск и сортировка (по дате, по алфавиту)
- Отправка выбранных документов

#### Глобальная страница отчётов
- Агрегирует выполненные задачи со всех проектов
- Форма отправки отчёта: выбор проекта, задачи, таймер, комментарий
- После отправки форма скрывается, отчёт появляется в истории
- Возврат задач из «Выполненных» обратно в активные

#### Сайдбар и навигация
- Проекты отображаются динамически в сайдбаре
- Кнопка «Создать новый» → модальное окно (только имя + описание)
- Отдельная вкладка «Отчёты» в навигации
- Топбар: кнопка «+» открывает меню: Задача / Заметка / Документ / Библиотека промтов
